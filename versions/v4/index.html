Layout stability fix included: the canvases now have fixed container heights and the animation loop only resizes the drawing buffer, not the CSS box. That removes the upward stretch / page-growth behavior.
What was causing the scroll-like stretch?

The earlier animation measured a canvas container, then wrote a new CSS width and height back to the canvas every frame. In a grid layout this can create a feedback loop where the content keeps affecting the container height.

What changed?

The visual containers now use fixed responsive heights such as height: clamp(...). JavaScript updates only canvas.width and canvas.height for sharp rendering. It does not set canvas.style.height during animation.

Location optimizer

The tool does not pretend to pick a purchasable parcel. Instead, it scores location archetypes and shows the parcel-level checks that must be proven: zoning district, special-use path, grid interconnection, water availability, floodplain, wetlands, endangered species, noise, generators, traffic, taxes, and local benefit.

Testing before and after construction

Water must be measured before the facility changes the site, during construction when sediment and spills are most likely, and after operation begins. The model treats water monitoring as a public accountability system, not a one-time report.
